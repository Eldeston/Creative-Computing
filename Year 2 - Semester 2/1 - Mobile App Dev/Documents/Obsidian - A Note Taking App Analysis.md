# Introduction

Obsidian. Obsidian is a free, powerful, and customizable note taking app that utilizes Markdown to write down notes. Note takers write their notes using the Markdown pipeline to write efficiently while offering great customization. Obsidian is available in iOS, Android, Mac, Windows, and Linux systems.

# Purpose - Who is it for?
## Targeted Audience

Obsidian can be for individuals who are new to note writing but want more customization. Obsidian can also be for programmers who've had experience with documentation with Markdown in Github.

## Problem

The problem with modern notetaking apps is that these notes can be easily lost when you move into a new device or platform. Users would have to move them manually before they completely move on to their new device. These apps often keep notes online, making them inaccessible offline. Another problem is that there was no universal file format for notes. In other words, you'd require a specific app to open and read the notes.

## Solution

Obsidian solves both these through Markdown. Markdown is an open text file format that can be accessed as plaintext. It can be accessed by any ordinary text editor. It also exists as a file that can be easily transferred via cloud and can be accessed in any device completely offline.

Obsidian also brings other features to the table. Key features such as its canvas view, command palette, and its graph view which gives users more ways to organize their notes and formulate ideas. Customization features are also available for themes and plugins made by Obsidian or its community.

# Aesthetics - How does it look?

Obsidian's aesthetics offers great customization thanks to its Markdown pipeline. Because of the Markdown pipeline, the texts can be easily styled and sized appropriately across different device layouts.

On top of this, it also offers great customization options for the user through downloadable themes made by Obsidian or creators from its community. The themes can be installed and used for each Obsidian vault allowing separate themes for separate workspaces. Obsidian offers all these customization options while also offering a plug-and-play setup.

# Architecture - What is its layout?

Obsidian maintains consistent aesthetics while optimizing for screen size in architecture for accessibility purposes. For tablets, while similar to desktop view, the file drawer is opened by default to get an overview of your notes and can be opened or closed while writing. For phones, the drawer is closed and the writing view is optimized to use the entire screen for immersion. Both devices are optimized for touch gestures.

## Note Menu

Obsidian is optimized to use a responsive layout for mobile, tablets, and desktop screens. For mobile devices, it features a drawer button to access all your notes. It also features reading mode and menu options for the note that you're editing in.

## Navigation Menu

Obsidian also features a navigation menu for your note that lets you search for keywords, seeing previously opened notes, and creating a new note. The navigation menu also lets you open/create other interfaces such as your canvas view, graph view, base view, your daily notes, inserting templates, or the command palette.

# Conclusion

I use Obsidian as a daily drive on my Windows laptop and my Android phone to write notes during class especially when it comes to documentation and programming. Overall Obsidian is a great tool for dumping all your ideas and organizing them later to form links and references with other ideas as well as remembering all the important points your professors made during class.

# Mobile App Evaluation Framework (Copilot Summary)
## Function — How well the app behaves during real use

This category evaluates whether the app works reliably and supports smooth interaction. It focuses on the _actual experience_ of using the app, not the code behind it.

- **Touch gestures respond correctly** — Taps, swipes, long‑presses, and scrolls behave as expected without lag or misfires.
- **App loads quickly** — Startup time is reasonable, and screens don’t freeze or hang.
- **Instructions or help available** — Tooltips, onboarding screens, or help sections guide new users.
- **No crashes or errors** — The app handles unexpected input gracefully and recovers without breaking.

Function is the baseline: if the app doesn’t work smoothly, everything else becomes irrelevant.
## Aesthetic — How the app looks and feels visually

This focuses on the visual design and emotional impression of the app. It aligns closely with Android’s Material Design principles.

- **Consistency across screens** — Colors, fonts, icons, and layouts follow a unified style.
- **Appropriate use of white space** — Screens feel balanced, not cramped or empty.
- **Graphics support content** — Images and icons clarify meaning instead of distracting.
- **Animations enhance experience** — Motion feels smooth and purposeful, not excessive or slow.

Aesthetic quality strongly influences user trust and perceived professionalism.
## Architecture — How the app is structured and organized

This is about the _layout and navigation structure_, not the internal code architecture. It evaluates how information and screens are arranged.

- **Number of screens** — The app avoids unnecessary screens and keeps flows efficient.
- **Navigation menu clarity** — Bottom nav, drawer, or tabs are intuitive and easy to understand.
- **Information hierarchy** — Important content is easy to find; secondary content is placed logically.

Good architecture reduces cognitive load and helps users move through the app effortlessly.
## Purpose — Why the app exists and who it serves

This category examines the conceptual foundation of the app—its value and relevance.

- **Who is it for?** — The target audience is clearly defined.
- **What problem does it solve?** — The app addresses a real need or pain point.
- **Why should anyone download it?** — The value proposition is clear and compelling.
- **Does it make life easier, faster, or more fun?** — The app provides meaningful benefit.

Purpose ensures the app isn’t just functional, but _worth using_.
## How this FAAP model works together

When reviewing an Android app, FAAP gives you a balanced lens:

- **Function** checks reliability.
- **Aesthetic** checks visual quality.
- **Architecture** checks usability and structure.
- **Purpose** checks relevance and value.

Together, they form a complete evaluation of both the user experience and the conceptual strength of the app.
